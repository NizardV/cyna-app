# Internationalisation (i18n)

## Approche adoptée

Le projet utilise le système de ressources **Android standard** (`res/values/strings.xml`) avec `stringResource()` dans les composables Compose.

---

## Structure des fichiers

```
app/src/main/res/
├── values/
│   └── strings.xml          ← Langue par défaut (français)
├── values-en/
│   └── strings.xml          ← Anglais
├── values-ar/
│   └── strings.xml          ← Arabe (RTL)
└── values-iw/
    └── strings.xml          ← Hébreu (RTL)
```

---

## Utilisation dans les composables

```kotlin
// Chaîne simple
Text(text = stringResource(R.string.login_title))

// Avec paramètre
Text(text = stringResource(R.string.greeting, userName))

// Dans un bouton
KButton(text = stringResource(R.string.login_button), onClick = onLogin)
```

## Ajouter une nouvelle chaîne

### 1. Ajouter dans `values/strings.xml` (français, référence)

```xml
<resources>
    <!-- … existant … -->
    <string name="catalog_title">Catalogue</string>
    <string name="catalog_search_placeholder">Rechercher un produit…</string>
    <string name="catalog_empty">Aucun produit trouvé.</string>
    <string name="catalog_product_count">%1$d produit(s)</string>
</resources>
```

### 2. Ajouter les traductions

```xml
<!-- values-en/strings.xml -->
<string name="catalog_title">Catalog</string>
<string name="catalog_search_placeholder">Search a product…</string>
<string name="catalog_empty">No products found.</string>
<string name="catalog_product_count">%1$d product(s)</string>
```

```xml
<!-- values-ar/strings.xml -->
<string name="catalog_title">الكتالوج</string>
```

### 3. Utiliser dans le composable

```kotlin
Text(text = stringResource(R.string.catalog_title))
KInput(placeholder = stringResource(R.string.catalog_search_placeholder))
Text(text = stringResource(R.string.catalog_product_count, state.products.size))
```

## Chaînes avec paramètres

```xml
<!-- Un paramètre -->
<string name="greeting">Bonjour, %1$s !</string>

<!-- Plusieurs paramètres -->
<string name="order_summary">Commande #%1$d — %2$s</string>
```

```kotlin
stringResource(R.string.greeting, user.firstName)
stringResource(R.string.order_summary, order.id, order.status)
```

## Support RTL (arabe, hébreu)

Le manifest déclare `android:supportsRtl="true"`. Les composants Compose gèrent le RTL automatiquement si les paddings et arrangements utilisent les attributs sémantiques :

```kotlin
// ✅ RTL-compatible
Modifier.padding(start = 16.dp, end = 8.dp)   // adapté selon la locale
Row(horizontalArrangement = Arrangement.Start)

// ❌ Direction forcée
Modifier.padding(left = 16.dp, right = 8.dp)
```

---

## Textes non traduits (hardcodés)

Certains textes sont intentionnellement hardcodés en anglais car ils sont :
- Noms propres de produits (`"Cyna EDR"`, `"Shield XDR"`)
- Labels techniques (`"PDF"`, `"Auto"`)
- Placeholders en cours de développement

Les statuts de commandes (`"Paid"`, `"Pending"`, `"Failed"`, `"Refunded"`) sont localisés via la fonction `localizeStatus()` dans `order/row.kt` plutôt que via des ressources strings, car ils correspondent à des valeurs d'enum .NET côté API.

## Convention de nommage des clés

```
{écran}_{élément}_{type}

Exemples :
login_email_label       → label du champ email sur l'écran login
login_button            → bouton principal de login
register_title          → titre de l'écran d'inscription
password_criteria_length → critère de longueur du mot de passe
header_menu_logout      → item "Déconnexion" dans le menu du header
error_invalid_email     → message d'erreur email invalide
```
